# Генератор Документов

Приложение для автоматического создания DOCX-документов на основе данных из Excel-файлов и Word-шаблонов. Позволяет загрузить шаблон Excel, применить настраиваемые правила трансформации данных и сгенерировать пакет документов по Word-шаблонам с поддержкой Jinja2-переменных.

**Стек:** Streamlit (веб-интерфейс) + FastAPI (REST API) + Python

---

## Возможности

- **Загрузка и валидация Excel** — парсинг шаблонов с проверкой данных, выявлением пропущенных полей и дубликатов
- **Трансформация данных** — применение настраиваемых правил преобразования (числа в слова, склонение имён, извлечение частей дат и др.)
- **Генерация DOCX** — создание документов из Word-шаблонов с Jinja2-переменными
- **Веб-интерфейс** — удобный UI на Streamlit с пошаговым рабочим процессом
- **REST API** — FastAPI-бэкенд для интеграции с внешними системами
- **Архивация результатов** — автоматическая упаковка сгенерированных документов в ZIP

pip install -e ".[web]"

## Запуск

### Способ 1: BAT-скрипт (Windows)

```bash
run-web.bat
```

### Способ 2: Ручной запуск

**Терминал 1 — FastAPI бэкенд:**

```bash
uvicorn src.web.api.main_api:app --reload --host 0.0.0.0 --port 8000
```

**Терминал 2 — Streamlit фронтенд:**

```bash
streamlit run src/web/frontend/app.py
```

Приложение будет доступно по адресу: **http://localhost:8501**

## Руководство по использованию

### Шаг 1: 📥 Загрузка данных

1. Откройте страницу **«Load Data»**
2. Загрузите Excel-файл (`.xlsx` / `.xls`)
3. Нажмите **«Analyze & Load»**
4. Проверьте результаты:
   - **📊 Loaded Data** — предпросмотр загруженных данных
   - **🗺️ Variable Mapping** — сопоставление переменных с читаемыми именами
   - **⚠️ Validation Errors** — ошибки и предупреждения валидации

> **Формат Excel:** первая строка — читаемые имена полей, вторая строка — имена переменных (для использования в шаблонах), начиная с третьей строки — данные.

### Шаг 2: 🔄 Трансформация данных

1. Перейдите на страницу **«Transform Data»**
2. Проверьте выбранные правила трансформации (загружаются из настроек)
3. Нажмите **«Transform Data»**
4. Проверьте результаты:
   - **🧩 Transformed Columns** — новые преобразованные столбцы
   - **🗺️ Variable Mapping** — обновлённое сопоставление переменных
   - **⚠️ Issues** — ошибки трансформации

### Шаг 3: 📄 Генерация документов

1. Перейдите на страницу **«Generate Documents»**
2. Загрузите ZIP-архив с Word-шаблонами (`.docx`)
3. Нажмите **«Generate Documents»**
4. Скачайте результат кнопкой **«Download Generated Documents (ZIP)»**
5. Проверьте статус генерации и возможные ошибки

> **Шаблоны Word** поддерживают Jinja2-переменные в формате `{{variable_name}}`. Имя файла шаблона и путь к выходному файлу также могут содержать переменные для динамической генерации имён и структуры папок.

### Шаг 4: ⚙️ Настройки

1. Откройте страницу **«Settings»**
2. Настройте правила трансформации:
   - Включите/выключите правила переключателем **«Selected»**
   - Настройте параметры каждого правила
   - Удаляйте ненужные правила кнопкой **❌**
   - Добавляйте новые правила через **«Add Rule»**
3. Нажмите **«Save All Settings»**

## API Эндпоинты

| Метод | Эндпоинт        | Описание                                              |
| ---------- | ----------------------- | ------------------------------------------------------------- |
| `GET`    | `/settings`           | Получение текущих настроек            |
| `POST`   | `/settings`           | Обновление настроек                         |
| `POST`   | `/process-excel`      | Загрузка и обработка Excel-файла       |
| `POST`   | `/transform-data`     | Трансформация данных по правилам |
| `POST`   | `/generate-documents` | Генерация DOCX-документов                  |
| `GET`    | `/download/{archive}` | Скачивание ZIP-архива документов    |

**Документация API** доступна по адресу: `http://localhost:8000/docs` (Swagger UI)

## Правила трансформации

Доступные типы правил:

| Правило               | Описание                                                                         |
| ---------------------------- | ---------------------------------------------------------------------------------------- |
| **NumToWordsRule**     | Преобразование чисел в текстовое представление |
| **NameDeclensionRule** | Склонение имён по падежам                                          |
| **DatePartGetterRule** | Извлечение частей даты (день, месяц, год)                |
| **NamePartGetterRule** | Извлечение частей ФИО (фамилия, имя, отчество)      |
| **NameShortenerRule**  | Сокращение имён (инициалы и др.)                                |

Настройки правил сохраняются в файле `config/data_transformation_rules.json`.

## Конфигурация

### `config/data_transformation_rules.json`

Файл хранения настроек правил трансформации. Создаётся автоматически при первом запуске API. Структура:

```json
{
  "rules_list": [
    {
      "selected": true,
      "type": "numtowords",
      "rule_type": "NumToWordsRule",
      "old_var_name": "amount",
      "old_header_name": "Сумма",
      "new_var_name": "amount_words",
      "new_header_name": "Сумма прописью"
    }
  ]
}
```

### Папки логов

- `logs/fastapi/` — логи FastAPI-бэкенда
- `logs/streamlit/` — логи Streamlit-фронтенда
